# 复刻基线维护原则（AlohaMini）

> AlohaMini 复刻学习项目的依赖管理与同步规则。
> 本文件是同步/升级/维护决策的**最高依据**，优先级高于任何 AI 记忆和临时讨论。
> 方法论移植自 `reBot-DevArm`，按 AlohaMini 实际情况（无 submodule）精简。

---

## 一、项目定位

**本仓库是复刻学习工程，不是单一软件包维护。**

**核心目标**：学懂并能复刻 AlohaMini；持续跟踪上游，同时维护一套可回滚、可复现、已验证的复刻基线。

**一句话原则**：追最新，但不盲目追最新；更新必须记录来源、原因、验证结果和回滚点。

### 与 reBot-DevArm 的关键差异

| | reBot-DevArm | AlohaMini（本项目） |
|--|--------------|-------------------|
| 依赖结构 | 1 主仓 + 4 **submodule** | **3 个并列独立仓**，无 submodule |
| 治理复杂度 | 高（submodule 指针 + .gitmodules 单源） | 低（各仓独立 fork，平铺） |
| umbrella 仓 | 主仓本身就是 fork | 独立「笔记/治理」仓，只装文档 |

### 三层基线结构（概念照搬）

```
sync/<repo>-<日期> 分支  ←  追最新、试同步（可随时丢弃）
       ↓ 验证通过
main 分支               ←  已验证的「最新集成状态」（滚动）
       ↓ 阶段性固化
baseline-<日期> tag     ←  阶段性可复现快照（永不变动）
```

## 二、核心规则

1. **核心依赖 fork 到自己账号**（`fanhao375`）：fork 不是为改源码，是为**锁定权 + 可访问性**（防上游 force-push / 删库 / 转私有）。
2. **当前为「只读起步」阶段**：三仓 origin 仍指 liyiteng，**尚未 fork**。决定锁定时 → fork 到 fanhao375，origin 改指 fork、liyiteng 设 upstream。`lerobot_alohamini` 是 fork-of-fork，需配两层上游（liyiteng → huggingface）。
3. **同步前必须先检查**：`git fetch upstream && git log HEAD..upstream/main --oneline`（每仓）。
4. **不同步也要记录**：每次检查上游，无论同步与否，都写 `上游更新记录/<日期>_同步详情.md`（来源 / 当前 commit / 上游 commit / 新增数 / 主要变化 / 是否同步 / 不同步原因+下次触发 / 同步验证结果）。
5. **升级走 sync 分支**：不在 main 上直接升级；sync 分支验证通过再合 main，然后打 `baseline-<日期>` tag。
6. **commit 分层**：策略变更 / 上游同步 / 学习笔记 / 操作日志 / 文档整理 分开提交，便于 bisect 与回滚。
7. **永不擅自 push / fork / 改 remote**：任何对 GitHub 的写操作（push / fork / 改 origin-upstream）都需用户明确确认。

## 三、三仓同步策略

| 仓库 | fork 状态 | 同步策略 | 触发条件 | 验证门槛 |
|------|----------|---------|---------|---------|
| `liyiteng/AlohaMini` | ⏸️ 未 fork（计划 fanhao375） | 看 diff，硬件资料为主 | 大阶段前 / 知道有官方更新 | 低（看 diff 没问题就同步） |
| `liyiteng/AM-ARM` | ⏸️ 未 fork（计划 fanhao375） | 看 diff，URDF/STEP 一致性 | 进入装配/仿真阶段 / 上游有更新 | 中（URDF 关节数/命名一致） |
| `liyiteng/lerobot_alohamini` | ⏸️ 未 fork（计划 fanhao375） | fork + sync 分支验证；⚠️两层上游 | 上游有更新 / 进入软件实操 | 中（能装 + import + 跑到连接阶段） |

## 五、AI 协作红线

**必须做**：同步前检查状态；检查到上游先记录再决策；升级走 sync 分支；不同步写明原因+下次触发；阶段性工作提醒更新文档。

**必须不做**：❌ 擅自 push/fork/改 remote ❌ 擅自升级（改 gitlink）❌ 只信 AI 记忆做决策 ❌ 多类工作混一个 commit。

**用户授权后可做**：fork 到 fanhao375 / 改 origin-upstream / push。
