## 1. 执行器：用的是「舵机」不是「电机」

- AlohaMini 全身用 **飞特 Feetech STS 系列总线舵机**（serial bus servo / 总线舵机），**没有一个无框力矩电机/直驱电机**。这是它能压到 ~$1000 自建的根本。
- 主力型号（权威见 `AlohaMini/AlohaMini2/docs/BOM.md`）：
  - **STS-3215**（ST-3215-C018，12V，1/345 减速比，~¥110）— 底盘 3 轮 + follower 双臂小关节
  - **STS-3095**（ST-3095-C002，12V，**95 kg·cm**，~¥345）— 升降轴 + 双臂动力大关节
  - **STS-3215**（ST-3215-C046，**7.4V**，1/147）— leader 主动臂（低压 + 低减速比，便于反驱手感）
  - **Pro 版** 加 **STS-3250** 工业级
- **18 自由度** = 双臂 6+1 ×2（14）+ 3 全向轮 + 1 升降。leader 那对臂（14 舵机）是遥操作输入设备，不计入机器人 DoF。

### 舵机 vs 力矩电机（对照 reBot-DevArm）

| | AlohaMini（舵机） | reBot-DevArm（电机） |
|--|------------------|---------------------|
| 本质 | 减速箱+位置反馈+驱动一体化模块，给角度自己转过去 | 无框/准直驱力矩电机，需外部驱动器，能真力控 |
| 通信 | 串行总线（SCS/STS，半双工 3 线菊花链） | CAN @ 1Mbps |
| 控制 | 以**位置控制**为主 | 力矩/MIT 模式，能重力补偿 |
| 电压 | 12V / 7.4V | 24V / 48V |
| 单价 | ¥110~¥345 | 几百~上千/个 |
| 谱系 | SO-ARM100/101 / LeRobot 一系 | 达妙 / RobStride 系 |

> 推论：AlohaMini 没有重力补偿那套力控玩法；但总线舵机**能读回位置/负载**，足以做 leader-follower 遥操作与数据采集 —— 正契合它面向 LeRobot 模仿学习的设计取向。

## 2. 控制架构：树莓派 host / PC client，可 USB 直连替代

- 官方架构 = **LeKiwi 的 Client/Host**：
  - **Host** = 移动底盘 + follower 双臂 + 算力（树莓派 5），骑在车上
  - **Client** = PC 工作站 + leader 双臂
  - 两者走 **WiFi**，所以机器人能无线满地跑。
- **Waveshare 总线舵机控制器本质是 USB↔串口(TTL) 桥**；树莓派只是「跑 lerobot、开串口、按 Feetech 协议收发」的那台电脑。

### 能不能不用树莓派、用长 USB 线直连 PC？——能，但分场景

- ✅ **桌面/固定场景**（遥操作、采数据，机器人不用移动）：把 Waveshare 适配器 USB 插到 PC，PC 多出串口（`COMx`/`/dev/ttyUSB*`），跑同样 lerobot 代码即可。这就是 SO-ARM/LeRobot 桌面标准玩法。
- ⚠️ 代价/坑：
  1. **移动性没了** —— 长 USB 把机器人拴在桌边（移动机器人本想无线）。
  2. **USB 线长**：无源 USB2.0 可靠 ~5m 封顶；更长要主动延长线/USB-over-Ethernet。容易触发串口时序问题（参考 reBot 踩过的 usbipd 2Hz / write timeout）。
  3. **多路 USB 聚合**：底盘 1 + 每臂 1 ≈ 3~4 路舵机串口 **＋ 5 个 USB 摄像头**。只驱动舵机一根线够；**带摄像头采数据**需车上加带电源 USB Hub 单线上行，5 路 720p 还吃带宽。
- 结论：**学习/调试先 USB 直连 PC 最省事**；要它真移动再装回树莓派走 WiFi。
- ❓待验证：`lerobot_alohamini` 默认 host/client 启动方式，以及能否单机(PC) 把 host+client 一起跑、直接 USB 驱动。

## 3. 生态与上游关系

- `liyiteng/AlohaMini`（硬件 monorepo，无 submodule）+ `liyiteng/AM-ARM`（AM-ARM200 臂）+ `liyiteng/lerobot_alohamini`（软件，**fork 自 huggingface/lerobot**）。
- AlohaMini2 的臂 = **AM-ARM200**（6+1 DoF）；AlohaMini1 的臂 = **SO-ARM100/101**（5+1 DoF）。
- Pi-0.5 训练走官方 **openpi**（非 lerobot 内置实现），见 `AlohaMini/examples/pi0.5_openpi/`：16/18 DoF 对齐（16 DoF 补虚拟关节凑 18，18 DoF 直通）。
- 软件唯一源头是 `lerobot_alohamini`（`AlohaMini/software/README.md` 明确把它指为 single source of truth）。
